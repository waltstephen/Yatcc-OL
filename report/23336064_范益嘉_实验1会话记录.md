### 第二步：环境搭建
- 安装依赖：flex、clang-18、llvm-18-dev、bison、lld-18

### 第三步：需求分析

- 从所有标准答案中提取全部 token 类型：
  ```
  find build/test/task1 -name "answer.txt" -exec cat {} \; | awk '{print $1}' | sort -u
  ```
- 确定需要支持的完整 token 集合（9个关键字、15个运算符、8个分隔符等）
- 使用 `cat -A` 分析输出格式中的精确空白字符

### 第四步：代码实现

**修改 lex.hpp**：添加所有缺失的枚举值
- 关键字：CONST, VOID, IF, ELSE, WHILE, BREAK, CONTINUE
- 运算符：MINUS, STAR, SLASH, PERCENT, LESS, GREATER, LESSEQUAL, GREATEREQUAL, EQUALEQUAL, EXCLAIMEQUAL, AMPAMP, PIPEPIPE, EXCLAIM
- 添加 mLastTokenEndCol 和 mLastTokenEndLine 字段用于 EOF 位置追踪

**修改 lex.cpp**：更新 kTokenNames 数组和 come() 函数
- kTokenNames 数组与枚举值一一对应
- come() 函数增加 EOF 位置特殊处理逻辑

**修改 lex.l**：核心词法规则
- 重新设计 COME 宏，同时处理列号追踪和 token 报告
- 添加所有关键字规则（放在标识符规则之前）
- 添加双字符运算符规则（放在单字符运算符之前）
- 添加十六进制常量匹配规则：`0[xX]{H}+{IS}?`
- 实现预处理指令解析：提取文件名和行号，设置 yylineno = line - 1
- 实现空白字符处理：水平空白更新列号和 LeadingSpace，换行重置列号和 StartOfLine

**修改 main.cpp**：修正输出格式
- 输出 `Loc=<file:line:col>` 位置信息
- 修正 StartOfLine/LeadingSpace 标志的制表符和空格格式

### 第五步：调试过程

**Bug 1：列号偏移**
- 现象：所有 token 的列号都比预期大
- 原因：COME 宏中 ADDCOL() 被调用了两次（规则中显式调用一次，COME 宏内部又调用一次）
- 修复：重构 COME 宏，移除规则中的显式 ADDCOL() 调用

**Bug 2：EOF 位置和标志错误**
- 现象：EOF token 带有 [StartOfLine] 标志，位置在文件末尾换行符之后
- 原因：clang 将 EOF 报告在最后一个真实 token 之后，不带 StartOfLine
- 修复：在 struct G 中增加 LastTokenEnd 追踪字段，EOF 时使用上一个 token 的结束位置
